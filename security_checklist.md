# 地铁智行 — 安全威胁应对清单

对照初步威胁清单（实验1）中的 6 条威胁，逐条验证应对措施。

## 威胁 1：行程数据网络窃听

| 项目 | 内容 |
|------|------|
| 威胁描述 | 公共 WiFi 下攻击者截获含起终点和时间的方案数据，获取用户出行轨迹 |
| 威胁类型 | 数据泄露 |
| 应对措施 | 全链路 HTTPS 加密（生产环境配置 TLS 证书）；敏感字段不通过 URL 传输（POST 方式可进一步降低泄露风险） |
| 当前状态 | **已设计** — FastAPI 支持 HTTPS，生产部署时配置 nginx + Let's Encrypt 或云服务 TLS |
| 验证方式 | 使用 Wireshark 抓包验证生产环境无明文传输 |

## 威胁 2：后台管理平台越权操作

| 项目 | 内容 |
|------|------|
| 威胁描述 | 攻击者通过篡改请求参数修改票价规则或插入虚假优惠 |
| 威胁类型 | 越权访问 / 数据篡改 |
| 应对措施 | Pydantic 模型严格校验输入类型和范围；JWT 认证确保操作者身份；管理接口需要额外角色检查 |
| 当前状态 | **部分实现** — 输入校验已到位（Pydantic）、JWT 认证已实现；管理后台和角色权限控制尚未开发 |
| 验证方式 | 尝试越权修改非本人数据 → 应返回 401/403 |

## 威胁 3：规划接口拒绝服务攻击

| 项目 | 内容 |
|------|------|
| 威胁描述 | 脚本短时间内发送大量复杂换乘查询，耗尽后端计算资源 |
| 威胁类型 | 拒绝服务 (DoS) |
| 应对措施 | slowapi 频率限制（规划接口 30次/分钟/IP）；RAPTOR 引擎 `max_transfers=5` 限制搜索深度；路径遍历深度受线路长度限制 |
| 当前状态 | **已实现** — `middleware/rate_limiter.py` + slowapi 装饰器 |
| 验证方式 | `test_security.py::test_rate_limit_on_plan` — 35次请求后返回 429 |

## 威胁 4：本地 SQLite 数据篡改

| 项目 | 内容 |
|------|------|
| 威胁描述 | 恶意用户在 Root 手机上修改本地 SQLite 线路权重，获得错误路线 |
| 威胁类型 | 数据篡改 |
| 应对措施 | 服务端 SQLite 仅在后端使用，客户端不可直接访问；后端为数据源权威方；生产环境可添加数据库文件哈希校验 |
| 当前状态 | **已缓解** — C/S 架构下客户端无法直接修改服务端数据；离线场景需添加签名校验 |
| 验证方式 | 哈希校验：启动时比对 SQLite 文件 SHA256 与预置签名 |

## 威胁 5：地图数据包 XSS 攻击

| 项目 | 内容 |
|------|------|
| 威胁描述 | 管理员导入含恶意脚本的站内地图矢量数据包，同步到客户端后触发 XSS |
| 威胁类型 | 恶意输入 / 跨站攻击 |
| 应对措施 | 前端使用纯 HTML/CSS/JS 渲染（非 WebView），不执行外部脚本；内容安全策略 (CSP) Header 限制脚本来源 |
| 当前状态 | **已缓解** — 当前 Web 前端无地图数据包导入功能，该功能未实现；若后续实现需添加数字签名验证 |
| 验证方式 | 尝试注入 `<script>` 标签内容 → 浏览器不执行或 CSP 拦截 |

## 威胁 6：接口数据爬取

| 项目 | 内容 |
|------|------|
| 威胁描述 | 攻击者伪造客户端批量调用开门方向接口，爬取全量数据 |
| 威胁类型 | 身份伪造 / 数据抓取 |
| 应对措施 | slowapi 频率限制（每 IP 限制）；JWT 认证限制敏感接口访问；生产环境可添加 App 签名验证 |
| 当前状态 | **已实现** — slowapi 全局 + 接口级频率限制；`/api/auth/me` 需 JWT 认证 |
| 验证方式 | 未登录访问 `/api/auth/me` → 403；高频访问 `/api/plan` → 429 |

---

## 总结

| 威胁编号 | 威胁名称 | 应对状态 |
|----------|----------|----------|
| 1 | 数据窃听 | 已设计（需生产环境部署 TLS） |
| 2 | 越权操作 | 部分实现（缺管理后台和 RBAC） |
| 3 | DoS 攻击 | **已实现**（slowapi 频率限制） |
| 4 | SQLite 篡改 | 已缓解（C/S 架构隔离） |
| 5 | XSS 攻击 | 已缓解（纯前端渲染、无外部脚本） |
| 6 | 数据爬取 | **已实现**（频率限制 + JWT） |

6 条威胁中：2 条已完整实现、2 条已有效缓解、1 条已设计待部署、1 条部分实现待完善。
